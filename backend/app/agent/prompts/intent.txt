Analise a solicitação do usuário e identifique:

1. **intent**: A intenção principal. Deve ser um dos valores:
   - create_rule: Criar nova regra de firewall/acesso
   - delete_rule: Deletar/remover uma regra de firewall/acesso existente
   - edit_rule: Editar/modificar uma regra de firewall/acesso existente
   - create_group: Criar grupo de endereços/serviços
   - list_rules: Listar regras de firewall/acesso existentes
   - list_nat_policies: Listar políticas de NAT
   - create_nat_policy: Criar nova política de NAT
   - delete_nat_policy: Deletar/remover uma política de NAT existente
   - list_route_policies: Listar políticas de rota/rotas estáticas
   - create_route_policy: Criar nova política de rota/rota estática
   - delete_route_policy: Deletar/remover uma política de rota existente
   - configure_content_filter: Configurar filtro de conteúdo/content filter (bloquear/liberar categorias de sites, criar perfil ou política de CFS)
   - toggle_gateway_av: Ativar ou desativar o Gateway Anti-Virus (GAV/antivírus de gateway)
   - toggle_anti_spyware: Ativar ou desativar o Anti-Spyware / anti-spyware
   - toggle_ips: Ativar ou desativar o IPS / Intrusion Prevention System / prevenção de intrusão
   - toggle_app_control: Ativar ou desativar App Control / controle de aplicativos, ou bloquear/liberar categorias de aplicações
   - toggle_geo_ip: Ativar ou desativar Geo-IP Filter / filtro geográfico, ou bloquear/liberar países
   - toggle_botnet: Ativar ou desativar Botnet Filter / filtro de botnet
   - toggle_dpi_ssl: Ativar ou desativar DPI-SSL (Deep Packet Inspection para SSL, inspeção SSL cliente ou servidor)
   - configure_app_rules: Criar ou configurar política de App Rules / regra de aplicação
   - add_security_exclusion: Adicionar endereço IP a grupo de exclusão (Geo-IP, Botnet ou CFS)
   - health_check: Verificar status/conectividade do dispositivo
   - get_snapshot: Obter snapshot da configuração atual
   - unknown: Solicitação não reconhecida

2. **present_fields**: Campos já fornecidos pelo usuário (ex: src_address, dst_address, service, action, name)

3. **missing_fields**: Campos obrigatórios que ainda não foram fornecidos

4. **extracted_data**: Dados já extraídos da solicitação em formato estruturado

Solicitação do usuário:
{user_input}
