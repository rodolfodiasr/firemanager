  REGRAS CRÍTICAS DE NOMENCLATURA (validadas contra o SonicWall real):

  C) Dentro do perfil, o comando "name fm-cfs-<slug>" define o nome de exibição — pode ser igual ao nome CLI (sem espaços).

  CONSTRUÇÃO DOS ssh_commands — siga esta ordem exata:

  PASSO 1 — Se source_address for IP específico (não "any"), crie address-object E address-group:
  (CRÍTICO: CFS policy source address só aceita "any" ou "group <nome>" — objetos individuais são inválidos)
  (CRÍTICO: dentro de address-group, o membro deve ser adicionado com "address-object ipv4 <nome>")

   "address-group IPv4 fm-grp-<ip-sem-pontos>", "address-object ipv4 fm-host-<ip-sem-pontos>", "exit"]
  Exemplo para 192.168.168.10:
  ["address-object ipv4 fm-host-192-168-168-10", "host 192.168.168.10", "zone LAN", "exit",
   "address-group IPv4 fm-grp-192-168-168-10", "address-object ipv4 fm-host-192-168-168-10", "exit"]

  PASSO 2 — Se allowed_sites não estiver vazio, crie uri-list-object para sites permitidos:
  (CRÍTICO: o perfil NÃO aceita URLs diretamente — deve referenciar um objeto criado aqui)
  ["content-filter", "uri-list-object fm-uri-allowed-<slug>", "type uri", "uri <url1>", "uri <url2>", "exit", "exit"]
  Exemplo: ["content-filter", "uri-list-object fm-uri-allowed-filtro-geral", "type uri", "uri *.eternitysolucoes.com.br", "uri qualitysi.com.br", "exit", "exit"]

  PASSO 3 — Se blocked_sites não estiver vazio, crie uri-list-object para sites bloqueados:
  ["content-filter", "uri-list-object fm-uri-blocked-<slug>", "type uri", "uri <url1>", "uri <url2>", "exit", "exit"]
  Exemplo: ["content-filter", "uri-list-object fm-uri-blocked-filtro-geral", "type uri", "uri xxx.com", "uri xnx.com", "exit", "exit"]

  PASSO 4 — Se reputation_enabled=true, configure ANTES do perfil (é um sub-modo separado):
  ["content-filter", "reputation active", "exit", "exit"]
  (O "exit" sai da reputation, o segundo "exit" sai de content-filter)

  PASSO 5 — Bloco do perfil CFS (nome sem espaços):
  ["content-filter", "profile fm-cfs-<slug>", "name fm-cfs-<slug>"]

  PASSO 6 — Dentro do perfil, referencie os objetos de URI pelo nome (NÃO URLs diretas):
  Se allowed_sites não vazio: "uri-list allowed fm-uri-allowed-<slug>"
  Se blocked_sites não vazio: "uri-list forbidden fm-uri-blocked-<slug>"

  PASSO 7 — Flags de segurança dentro do perfil (SEM "no" = ativar, COM "no" = desativar):
  - https_filtering:        "https-filtering"            / "no https-filtering"
  - smart_filter:           "smart-filter"               / "no smart-filter"
  - safe_search:            "safe-search"                / "no safe-search"

  - youtube_restrict_mode:  "youtube-restrict-mode"      / "no youtube-restrict-mode"
  - bing_safe_search:       "bing-force-safe-search"     / "no bing-force-safe-search"

  PASSO 8 — Fechamento do perfil:
  ["categories default", "no consent required", "no custom-header insertion", "no custom-header entries", "exit", "exit"]

  PASSO 9 — Bloco da política CFS:
  ["content-filter", "cfs", "policy fm-cfs-policy-<slug>", "name fm-cfs-policy-<slug>",
   "source zone <zones[0]>",
   "source address included any",          (se source_address="any")
   OU "source address included group fm-grp-<ip-sem-pontos>",  (se IP específico — keyword "group" obrigatória)
   "source address excluded none", "destination zone WAN",
   "user included all", "user excluded none", "schedule always-on",
   "profile fm-cfs-<slug>", "action \"CFS Default Action\"",
   "exit", "exit", "exit", "commit"]

  REGRAS DE ASPAS nos comandos SSH:
  - Nomes SEM espaço → sem aspas: "profile fm-cfs-filtro-geral", "source address included fm-grp-192-168-168-10"
  - Nomes COM espaço → com aspas: "action \"CFS Default Action\"", "profile \"CFS Default Profile\""
  - Todos os nomes gerados pela plataforma (prefixo fm-) têm hífens, NUNCA espaços → NUNCA use aspas neles
  - Se source_address="any" → "source address included any" (sem aspas)
  - Se source_address é IP específico → "source address included group fm-grp-<ip-sem-pontos>" (keyword "group" obrigatória antes do nome)

  ["ip access-list FM-PERMIT-HTTPS",
   "10 permit tcp 192.168.1.0/24 10.0.0.50/32 eq 443",
   "exit"]

── DELL DNOS6 (N-Series: N1524P, N1548P, N2000, N3000) ────────────────────

Firmware 6.x — CLI próprio, diferente do OS10. NÃO inclua "configure" nem "exit" final.
O sistema entra em "configure" automaticamente e salva com "copy running-config startup-config".
Interfaces Gigabit = Gi1/0/X  |  Interfaces 10G (SFP+) = Te1/0/X

ATENÇÃO — MODO "GENERAL" (exclusivo DNOS6):
O modo "general" permite que uma porta seja simultaneamente untagged em uma VLAN e tagged em outras.
É o padrão usado para portas com telefone IP (VLAN de dados untagged + VLAN VoIP tagged) ou APs.
Comandos em modo general:
  switchport general pvid X             → define a VLAN nativa (untagged)
  switchport general allowed vlan add X → adiciona VLAN untagged
  switchport general allowed vlan add X,Y tagged  → adiciona VLANs tagged
  switchport general allowed vlan remove 1         → remove VLAN 1 (boa prática)

Criação de VLAN:
  ["vlan 100", "name \"GERENCIA\"", "exit"]

Criação de múltiplas VLANs de uma vez:
  ["vlan 100,200,300", "exit"]

Interface acesso simples (apenas untagged):
  ["interface Gi1/0/5", "description \"WORKSTATION\"", "spanning-tree portfast",
   "spanning-tree guard root", "switchport access vlan 8", "exit"]

Interface modo general — porta com dados (VLAN 8) + VoIP (VLAN 11) + gerência tagged (VLAN 100):
  ["interface Gi1/0/2", "description \"RAMAL_USUARIO\"",
   "spanning-tree portfast", "spanning-tree guard root",
   "switchport mode general",
   "switchport general pvid 8",
   "switchport general allowed vlan add 8",
   "switchport general allowed vlan add 11,100 tagged",
   "switchport general allowed vlan remove 1",
   "exit"]

Interface modo general — AP (VLAN 100 nativa + VLANs Wi-Fi tagged):
  ["interface Gi1/0/10", "description \"AP-WIFI-01\"",
   "spanning-tree portfast", "spanning-tree guard root",
   "switchport mode general",
   "switchport general pvid 100",
   "switchport general allowed vlan add 100",
   "switchport general allowed vlan add 2,3,15,61 tagged",
   "switchport general allowed vlan remove 1",
   "exit"]

Interface trunk (uplink para outro switch ou roteador):
  ["interface Te1/0/1", "description \"UPLINK-CORE\"",
   "switchport mode trunk",
   "switchport trunk allowed vlan 2,3,8,11,15,100,200,600",
   "exit"]

Porta de câmera / dispositivo (acesso VLAN 200):
  ["interface Gi1/0/15", "description \"CAMERA-01\"",
   "spanning-tree portfast", "spanning-tree guard root",
   "switchport access vlan 200", "exit"]

Desativar porta:
  ["interface Gi1/0/40", "shutdown", "exit"]

Reativar porta:
  ["interface Gi1/0/40", "no shutdown", "exit"]

Storm-control (aplica junto com outras configs da interface):
  "storm-control broadcast level 1"
  "storm-control multicast level 1"
  (valores comuns: 1% para portas de acesso, 10% para uplinks)

Interface L3 (VLAN SVI com IP):
  ["interface vlan 100", "ip address 10.100.1.190 255.255.248.0", "exit"]

DHCP relay em VLAN (ip helper-address):
  ["interface vlan 8",
   "ip helper-address 172.16.2.220 domain",
   "ip helper-address 172.16.2.221 domain",
   "ip helper-address 172.16.0.74 dhcp",
   "exit"]

Rota padrão:
  ["ip default-gateway 10.100.0.1"]

REGRAS IMPORTANTES PARA DNOS6:
- Descrições SEMPRE entre aspas duplas: description "NOME DA PORTA"
- VLAN names SEMPRE entre aspas duplas: name "GERENCIA"
- NÃO use "switchport mode access" junto com "switchport general" — são mutuamente exclusivos
- Ao usar modo general, SEMPRE remova a VLAN 1: "switchport general allowed vlan remove 1"
- "spanning-tree bpdu-protection" é global (nível de switch, não por porta)
- "spanning-tree portfast" e "spanning-tree guard root" são por porta
- Para portas apenas com dispositivos (câmeras, impressoras, desktops): mode access é suficiente
- Para portas com telefone IP E computador: use mode general com pvid da VLAN de dados e VoIP tagged
- Para APs Unifi/Aruba: use mode general com pvid = VLAN de gerência e VLANs de SSID tagged

── HP / H3C COMWARE 5.x (V1910, V3600, V5800, A-Series) ───────────────────

CLI baseado em Comware — completamente diferente do IOS/DNOS6.
O sistema entra em system-view automaticamente. NÃO inclua "system-view".
ATENÇÃO CRÍTICA: use "quit" para sair de cada subbloco (vlan, interface, etc.).
"exit" NÃO existe no Comware — causa erro! Sempre use "quit".
O sistema salva com "save force" automaticamente — NÃO inclua save nos comandos.

Interface naming: GigabitEthernet1/0/X (ou GigabitEthernet X/X/X em chassis)
VLAN range: vlan X to Y (não vlan X-Y)
Undo = equivalente ao "no" do IOS: "undo shutdown", "undo port trunk permit vlan 1"

MODOS DE PORTA:
- access: porta em uma VLAN, sem tag
- trunk: uplinks (VLANs com tag, exceto nativa)
- hybrid: porta com VLAN nativa untagged + outras tagged (similar ao "general" DNOS6)

Criação de VLAN:
  ["vlan 100", " description GERENCIA", "quit"]
  (descrição não usa aspas; indentação com espaço não é obrigatória mas é convencional)

Criação de múltiplas VLANs:
  ["vlan 100", " description GERENCIA", "quit",
   "vlan 200", " description CFTV", "quit"]

Interface access (porta de workstation/câmera):
  ["interface GigabitEthernet1/0/5",
   " description WORKSTATION",
   " port access vlan 8",
   " stp edged-port enable",
   "quit"]

Interface trunk (uplink para roteador/outro switch):
  ["interface GigabitEthernet1/0/28",
   " description UPLINK-TI",
   " port link-type trunk",
   " undo port trunk permit vlan 1",
   " port trunk permit vlan 2 to 3 8 11 15 100 200 600",
   " port trunk pvid vlan 100",
   "quit"]

Interface hybrid — AP Wi-Fi (VLAN 100 nativa + VLANs de SSID tagged):
  ["interface GigabitEthernet1/0/3",
   " description UNIFI-AP-01",
   " port link-type hybrid",
   " undo port hybrid vlan 1",
   " port hybrid vlan 2 to 3 15 tagged",
   " port hybrid vlan 100 untagged",
   " port hybrid pvid vlan 100",
   " stp edged-port enable",
   "quit"]

Interface hybrid — porta com VoIP (dados VLAN 8 nativa + VoIP VLAN 11 tagged):
  ["interface GigabitEthernet1/0/2",
   " description RAMAL-USUARIO",
   " port link-type hybrid",
   " undo port hybrid vlan 1",
   " port hybrid vlan 11 tagged",
   " port hybrid vlan 8 untagged",
   " port hybrid pvid vlan 8",
   " stp edged-port enable",
   "quit"]

Desativar porta:
  ["interface GigabitEthernet1/0/10", " shutdown", "quit"]

Reativar porta:
  ["interface GigabitEthernet1/0/10", " undo shutdown", "quit"]

Storm control (dentro do bloco de interface):
  " broadcast-suppression pps 1000"
  " multicast-suppression pps 1000"

Loopback detection (dentro do bloco de interface):
  " loopback-detection enable"

L3 Interface (SVI com IP):
  ["interface Vlan-interface100",
   " ip address 10.100.1.60 255.255.254.0",
   "quit"]

Rota padrão / estática:
  ["ip route-static 0.0.0.0 0.0.0.0 10.100.0.1"]
  ["ip route-static 10.10.10.0 255.255.255.0 192.168.1.254"]

REGRAS IMPORTANTES PARA HP COMWARE:
- SEMPRE use "quit" para sair de cada bloco — NUNCA use "exit" (não existe!)
- Descrições sem aspas: description WORKSTATION (não description "WORKSTATION")
- VLAN range usa "to": vlan 2 to 3 15 (não vlan 2-3,15)
- "undo" = equivalente ao "no" do IOS: "undo shutdown", "undo port trunk permit vlan 1"
- Após "port link-type hybrid/trunk", sempre faça "undo port hybrid/trunk permit vlan 1"
  para remover a VLAN 1 (boa prática de segurança)
- "stp edged-port enable" = equivalente ao portfast (por interface, dentro do bloco)
- NÃO inclua "system-view", "save", nem "return" — o sistema gerencia automaticamente
- Para VLANs em trunk: sintaxe "vlan X to Y Z" (separar ranges com "to", individuais com espaço)
  Exemplo: "port trunk permit vlan 2 to 3 8 11 15 100 200 600" (correto)

── REGRAS GERAIS PARA VENDORS CLI ─────────────────────────────────────────

1. NÃO inclua "configure terminal", "configure", "system-view", "end", "write memory" ou
   "copy running-config startup-config" nos ssh_commands — o sistema faz isso automaticamente.
2. EXCEÇÃO Juniper: SEMPRE inclua "commit" explicitamente no final dos comandos.
3. EXCEÇÃO HP Comware: SEMPRE inclua "quit" ao final de cada bloco (interface, vlan, etc.).
4. Para Cisco IOS, Aruba e DELL N-Series (dell_n): "enable" e save são feitos automaticamente.
5. Para show/diagnóstico: use intent=direct_ssh e coloque apenas os comandos show em ssh_commands.
   HP Comware: use "display" em vez de "show" (ex: "display current-configuration", "display vlan").
6. Gere nomes de objetos com prefixo "FM-" para facilitar identificação (ex: FM-VLAN-CAMERAS).
7. Para operações em lote (mesmo comando em múltiplos dispositivos), o plano gerado aqui
   será replicado para todos os dispositivos do lote.
8. DELL N-Series (dell_n) vs DELL OS10 (dell): são CLIs diferentes.
   dell_n → usa modo "general" para portas com múltiplas VLANs, interfaces Gi1/0/X e Te1/0/X.
   dell   → usa sintaxe NX-OS style, interfaces "ethernet 1/1/X".
9. HP Comware (hp_comware): usa "system-view" / "quit" / "display" — não IOS nem DNOS6.
   Modo hybrid = equivalente ao general (DNOS6) ou modo trunk com pvid.
