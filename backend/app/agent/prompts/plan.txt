Regras de preenchimento:
- Para create_rule: preencha rule_spec com todos os campos, demais specs=null
- Para edit_rule: coloque o nome ou ID da regra em raw_intent_data["rule_id"]; em rule_spec preencha APENAS os campos que o usuário quer alterar — deixe os demais campos como "Any" ou vazio para que o sistema preserve os valores atuais; demais specs=null
- Para create_nat_policy: preencha nat_spec, demais specs=null
- Para create_route_policy: preencha route_spec, demais specs=null
- Para delete_nat_policy: coloque rule_id em raw_intent_data, specs=null
- Para delete_route_policy: coloque rule_id em raw_intent_data, specs=null
- Para delete_rule: coloque rule_id em raw_intent_data, specs=null
- Para create_group: preencha group_spec, demais specs=null
- Para list_rules: specs=null; em raw_intent_data coloque:
  - src_zone/dst_zone se o usuário mencionar zona de origem/destino
  - filter_any: true se o usuário pedir regras com objeto "any" ou "qualquer" em origem ou destino
  - filter_action: "allow" ou "deny" se o usuário pedir por ação específica
  - filter_enabled: true/false se o usuário pedir regras ativas/inativas
  - filter_object: "<nome do objeto>" se o usuário pedir regras que contenham um objeto específico em origem, destino ou serviço
- Para list_nat_policies: specs=null; em raw_intent_data coloque:
  - filter_object: "<nome>" se o usuário pedir NATs que contenham um objeto específico em origem, destino ou serviço
  - filter_inbound: "<interface>" se o usuário pedir por interface de entrada (ex: X1)
  - filter_outbound: "<interface>" se o usuário pedir por interface de saída (ex: X0)
  - filter_enabled: true/false se o usuário pedir NATs ativos/inativos
- Para list_route_policies: specs=null; em raw_intent_data coloque:
  - filter_object: "<nome>" se o usuário pedir rotas que contenham um objeto específico em origem, destino ou gateway
  - filter_interface: "<interface>" se o usuário pedir rotas por interface (ex: X1, X0)
  - filter_type: "<tipo>" se o usuário pedir por tipo de rota (ex: standard)
- Para configure_content_filter: defina execution_mode="ssh"; preencha content_filter_spec com os dados do filtro; em ssh_commands forneça APENAS os comandos de configuração (NÃO inclua "configure" nem "end" — o sistema entra e sai do modo configure automaticamente). Inclua "commit" ao final para salvar.

  IMPORTANTE — estrutura correta dos comandos CLI SonicWall (baseada no output real do dispositivo):
  "content-filter" é um MODO (como "configure") — deve ser enviado sozinho, sem argumentos.
  Dentro do modo content-filter, use "profile <nome>" para criar/editar perfil e "cfs" para entrar no modo CFS.
  Dentro do modo CFS, use "policy <nome>" para criar/editar política.
  Cada bloco encerra com "exit". Use dois "exit" para sair do profile (profile → content-filter → config) e três para policy (policy → cfs → content-filter → config).

  Exemplo para bloquear categorias padrão (adult, gambling) na zona LAN:
  ["content-filter", "profile fm-cfs-adult", "name fm-cfs-adult", "uri-list search-order allowed-first", "uri-list forbidden-operation block", "no reputation active", "categories default", "no https-filtering", "no smart-filter", "no safe-search", "no threat-api", "no google-force-safe-search", "no youtube-restrict-mode", "no bing-force-safe-search", "no consent required", "no custom-header insertion", "no custom-header entries", "exit", "exit", "content-filter", "cfs", "policy fm-cfs-policy-adult", "name fm-cfs-policy-adult", "source zone LAN", "source address included any", "source address excluded none", "destination zone WAN", "user included all", "user excluded none", "schedule always-on", "profile fm-cfs-adult", "action \"CFS Default Action\"", "enable", "priority 10", "exit", "exit", "exit", "commit"]

  Use nomes com prefixo "fm-" para perfis e políticas criados pela plataforma. Demais specs=null.

- Para toggle_gateway_av: defina execution_mode="ssh"; preencha security_service_spec com service="gateway-antivirus" e enabled=true/false; em ssh_commands gere os comandos abaixo (NÃO inclua "configure" nem "end"):
  Ativar:   ["gateway-antivirus", "enable", "exit", "commit"]
  Desativar: ["gateway-antivirus", "no enable", "exit", "commit"]
  Demais specs=null.

  Ativar:   ["anti-spyware", "enable", "exit", "commit"]
  Desativar: ["anti-spyware", "no enable", "exit", "commit"]
  Demais specs=null.

- Para toggle_ips: defina execution_mode="ssh"; preencha security_service_spec com service="intrusion-prevention" e enabled=true/false; em ssh_commands:
  Ativar:   ["intrusion-prevention", "enable", "exit", "commit"]
  Desativar: ["intrusion-prevention", "no enable", "exit", "commit"]

- Para toggle_app_control: defina execution_mode="ssh"; preencha security_service_spec com service="app-control", enabled e category_ids/category_block quando o usuário mencionar categorias; em ssh_commands:
  Ativar app-control geral: ["app-control", "enable", "exit", "commit"]
  Desativar app-control geral: ["app-control", "no enable", "exit", "commit"]
  Bloquear categoria 22: ["app-control", "category id 22", "block", "exit", "exit", "commit"]
  Liberar categoria 22: ["app-control", "category id 22", "no block", "exit", "exit", "commit"]
  Demais specs=null.

- Para toggle_geo_ip: defina execution_mode="ssh"; preencha security_service_spec com service="geo-ip", enabled e countries/country_action quando mencionados; em ssh_commands:
  Ativar geo-ip: ["geo-ip", "block connections", "exit", "commit"]
  Desativar geo-ip: ["geo-ip", "no block connections", "exit", "commit"]
  Bloquear país: ["geo-ip", "block country <País>", "exit", "commit"]
  Desbloquear país: ["geo-ip", "no block country <País>", "exit", "commit"]
  Múltiplos países: inclua um "block country <País>" por país na lista.
  Nome do país em inglês conforme SonicWall (ex: "China", "Russia", "North Korea").
  Demais specs=null.

- Para toggle_botnet: defina execution_mode="ssh"; preencha security_service_spec com service="botnet" e enabled=true/false; em ssh_commands:
  Ativar:   ["botnet", "block connections enable", "exit", "commit"]
  Desativar: ["botnet", "no block connections", "exit", "commit"]
  Demais specs=null.

- Para toggle_dpi_ssl: defina execution_mode="ssh"; preencha security_service_spec com service="dpi-ssl-client" ou "dpi-ssl-server" conforme o usuário mencionar; em ssh_commands:
  Ativar DPI-SSL cliente:   ["dpi-ssl client", "enable", "exit", "commit"]
  Desativar DPI-SSL cliente: ["dpi-ssl client", "no enable", "exit", "commit"]
  Ativar DPI-SSL servidor:   ["dpi-ssl server", "enable", "exit", "commit"]
  Desativar DPI-SSL servidor: ["dpi-ssl server", "no enable", "exit", "commit"]
  Se usuário mencionar "ambos" ou "client e server": gere dois blocos combinados (client e server) em um só ssh_commands com um único "commit" ao final.
  Demais specs=null.

- Para configure_app_rules: defina execution_mode="ssh"; preencha app_rules_spec com os dados da política; gere um nome para match-object com prefixo "fm-mo-" e para a policy com prefixo "fm-ar-"; em ssh_commands:
  Exemplo para bloquear uma lista de apps (IDs 100, 200) na zona LAN com Reset/Drop:
  ["match-object fm-mo-<nome>", "type application-list", "application id 100", "application id 200", "exit", "app-rules", "policy fm-ar-<nome>", "name fm-ar-<nome>", "enable", "type app-control", "source address any", "destination address any", "match-object included fm-mo-<nome>", "action-object Reset/Drop", "zone name LAN", "exit", "exit", "commit"]
  Para application-category-list, substitua "application id" por "application-category id".
  Demais specs=null.

- Para add_security_exclusion: defina execution_mode="ssh"; preencha security_exclusion_spec com group e ip_addresses; o grupo deve ser:
  - "GRP-EXCLUSION-GEO-IP" para exclusão de geo-ip
  - "GRP-EXCLUSION-BOOTNET" para exclusão de botnet (atenção: "BOOTNET" não "BOTNET")
  - "GRP-EXCLUSION-CFS" para exclusão de content filter
  Para cada IP, crie um address-object com prefixo "fm-excl-" e adicione ao grupo; em ssh_commands:
  Exemplo para adicionar 1.2.3.4 ao grupo GRP-EXCLUSION-GEO-IP:
  ["address-object ipv4 fm-excl-1-2-3-4", "name fm-excl-1-2-3-4", "zone LAN", "host 1.2.3.4", "exit", "address-group ipv4 GRP-EXCLUSION-GEO-IP", "address-object ipv4 fm-excl-1-2-3-4", "exit", "commit"]
  Para múltiplos IPs: repita o bloco address-object para cada IP antes do address-group, depois adicione todos no address-group com um único commit ao final.
  Substitua pontos por hífens no nome do objeto (ex: 10.0.0.1 → fm-excl-10-0-0-1).
  Demais specs=null.

- Para get_security_status: defina execution_mode="ssh"; em ssh_commands liste os comandos show desejados (sem "configure", sem "end"). O sistema executa em modo user exec (sem entrar em configure). Se o usuário não especificar quais serviços, inclua todos:
  ["show gateway-antivirus", "show anti-spyware", "show intrusion-prevention", "show app-control", "show geo-ip", "show botnet", "show dpi-ssl"]
  Se o usuário pedir apenas um serviço específico, inclua só o comando correspondente. Demais specs=null.
